Routine Read_LwRTM

Documentation:

Reads the longwave Radiative Transfer Model (atmospheric) file and loads appropriate data arrays. The supplied module reads the binary RTM data file.

The profile information is stored in a separate file.

Lw RTM file contents and "layout"

Header:

Instrument name
Date (YYYYMMDDHH)
Number of lat and lon points in grid
Number of pressure levels in grid and number of thermal channels for which data is stored
Instrument channel IDs (integer, e.g. 7   6   5)
Channel wavenumbers

Data blocks:

For each (lat, lon), a data block containing:
lat and lon values (plus one other value - not used: is always 0.0 in the current example file)
emissivity value for each channel
for each pressure level and channel:
    pressure level index      channel index (not ID)   Tbc   Tac   Rbc_up    Rac_up    Rac_dwn

e.g. 1 1 0.87692 1.00000 8.8495e+01 0.0000e+00 0.0000e+00

Channel varies fastest, e.g.
  1 1 0.87692 1.00000 8.8495e+01 0.0000e+00 0.0000e+00
  1 2 0.93153 1.00000 7.6555e+01 0.0000e+00 0.0000e+00
  1 3 0.91273 1.00000 1.6230e-01 0.0000e+00 0.0000e+00
  2 1 0.87698 0.99993 8.8498e+01 3.1008e-03 3.1008e-03

N.B. pressure level 1 corresponds to the lowest pressure value

Profile file contents and layout

Date
Number of lat and lon points in grid
For each lat, lon location:
    latitude and longitude values and number of pressure levels
    for each pressure level, in ascending order of pressure value:
         pressure and temperature values (plus other data - not used)
