Routine:  Read_SwRTM

Documentation:

Reads the shortwave Radiative Transfer Model (atmospheric) binary file and loads appropriate data arrays.

Unlike the supplied longwave RTM which is on a latitude, longitude grid; the supplied shortwave RTM results are based on three broad latitude bands (essentially the Tropical, Midlatitude and High latitude LOWTRAN7 atmpospheres) the central values of which are given in the file. (The accompanying shortwave part of Get_RTM is therefore an interpolation in latitude only.)

Data is therefore in the form T( level, channel, latitude_band )

Sw RTM file contents and layout

Header:

Instrument name
Number of lat band profiles
Latitudes at band centres
Number of pressure levels in grid
Number of solar channels for which data is stored
Instrument channel IDs (integer, e.g. 2   3   4)

Data blocks:

For each latitude band, a data block containing:
Latitude band header text (e.g. "Latitude band 1")
Above-cloud transmittance block header text (e.g. "Level to space transmittances (Tac):")
for each pressure level:
    pressure value      Tac (chan 1)    Tac (Chan 2)    etc
 
e.g. 100.0    1.0000    1.0000    1.0000
Below-cloud transmittance block header text (e.g. "Surface to Level transmittances (Tbc):")
for each pressure level:
    pressure value      Tbc (chan 1)    Tbc (Chan 2)    etc
